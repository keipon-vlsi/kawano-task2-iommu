# 面積・電力エスティメータ 出力例（estimate_report.md）

`cd iommu_sim && python3 run.py` の出力から、代表的な2構成（A: キャッシュ無し /
B: PWC + コアレッシング）を抜粋した例。**sky130 SEED 仮値による一次近似**であり、
相対比較・較正用（絶対サインオフではない）。詳細は `ESTIMATOR_ja.md` 参照。

  （メモリアクセス削減により）。面積はキャッシュ追加で増えるが、エネルギー効率は改善。

## スイープ要約（性能 + 面積/電力）

```
scenario     mem/pg  peakN   avg_ns   area_mm2   power_mW   pJ/xlate
--------------------------------------------------------------------
A             3.000      8    300.0   0.066248    12.8241    525.690
B             0.127      1     24.6   0.151538     6.5921    269.981
C             0.128      3      2.7   0.159538     6.6008    270.338
D             2.016      8    201.6   0.179538    11.2178    459.705
E             3.000      4    300.0   0.040648     6.7225    504.292
```

**読み方**：A→B でキャッシュを足すと面積は増えるが、メモリ往復が激減して
電力・エネルギー効率が改善。C は B にプリフェッチを足し遅延を 2.7 ns まで短縮
（エネルギーはほぼ同等）。D（ランダム IOVA）は局所性が崩れ、B の最適化効果が
消失して A 並みのエネルギーに戻る。E は有限リソース（walker=4, buffer=4）で
スループットが約半分に低下し、シム時間が延びるぶん pJ/xlate は高め。

各構成の凍結 JSON は `iommu_sim/freeze/*.json` に出力される（`config_hash` 付き、
後段の合成結果との突合用）。
